# Contributing

Contributions are welcome.

## Quick Start

To get started, do the following:

1. Fork this repository.
2. Clone your fork: `git clone https://github.com/your-username/llm-x.git`
3. Create a branch: `git checkout -b fix-something-cool`
4. Make the desired changes in the codebase.
5. Test it.
6. Commit with something real: `git commit -m "fix kv cache calc for MoE models"`
7. Push: `git push origin fix-something-cool`
8. Open a pull request against ```main```.

### Things to keep in mind

1. Ensure that the code is readable, maintainable, and extensible.
2. Make sure to make subsequent changes to ```README.md``` if a change made to the codebase impacts functionality.

### PR Guidelines

- Title like: ```feat: add support for GGUF (llama.cpp)```
- If your pull request is very large, open an issue to discuss it.
